Core Equation

The normalized small-signal system is written as:

\[ M(\omega)\,Y(\omega)=N(\omega),\qquad Y(\omega)=M^{-1}(\omega)\,N(\omega). \]

\(Y=[r,\phi,T_1]^T\) are demodulated normalized states and \(N\) is the source vector.

Role of \(M(\omega)\)

\(M(\omega)\) is the dynamic coupling operator that includes resonator response, single-island thermal response, electrothermal coupling, and optional linear detuning feedback.

  • Diagonal terms set intrinsic response poles/scales.
  • Off-diagonal terms encode coupling between \(r,\phi,T_1\).
  • Detuning enters through model coefficients and changes phase slope/responsivity.

Every source/noise path is computed through this same operator via \(Y=M^{-1}N\).

Implementation Form

In code, \(M(\omega)\) is built directly from project parameters and evaluated at demodulated frequency \(f\). The current implementation uses the single-KID state \([r,\phi,T_1]\).

Full \(M(\omega)\) Matrix Form Used in Code

Let \(\omega_0=2\pi f_{0,\mathrm{eff}}\), demodulated angular frequency \(\omega\), loaded \(Q\equiv Q_r\), and dimensionless operating detuning \(x\equiv \delta f/f_{0,\mathrm{eff}}\). The open-loop single-KID matrix is:

\[ M(\omega)= \begin{bmatrix} \dfrac{2i\omega Q_i}{\omega_0}+\dfrac{Q_i}{Q}+\beta_A+4Q_iQx^2 & -\dfrac{4i\omega Q_iQx}{\omega_0} & -2Qx\beta_A+\dfrac{2\alpha_{A1}}{T_{01}}\\[1.0em] \dfrac{4i\omega Q_iQx}{\omega_0}-\beta_\phi & \dfrac{2i\omega Q_i}{\omega_0}+\dfrac{Q_i}{Q}+4Q_iQx^2+2Qx\beta_\phi & -\dfrac{2\alpha_{\phi1}}{T_{01}}\\[1.0em] -\left(1+\dfrac{\beta_A}{2}\right)P_1 & Qx(\beta_A+2)P_1 & i\omega C_1+G_1-\dfrac{P_1\alpha_{A1}}{T_{01}} \end{bmatrix}. \]

Here \(P_1=P_0\) in the single-KID model.

A small commanded drive-frequency offset is represented as \(dx=df/f_0\), expanded around the operating value \(x=x_0+dx\). In the sideband basis this is a first-order carrier-frequency perturbation. Applying the Eq. (12) demodulation/normalization transform used throughout the code gives a phase-like normalized source column:

\[ B_x=\begin{bmatrix}0\\4iQ_i\\0\end{bmatrix}. \]

The sign follows the project convention \(x=\mathrm{detuning\_Hz}/f_0\); reversing the definition of detuning reverses this column.

With linear PID detuning feedback from the phase output,

\[ H_f(\omega)=K_f\left(1+\frac{1}{i\omega\tau_i}+ \frac{i\omega\tau_d}{1+i\omega\tau_d/N_d}\right),\qquad H_x(\omega)=\frac{H_f(\omega)}{f_0},\qquad C_\phi=\begin{bmatrix}0&1&0\end{bmatrix}, \]

Branches with nonpositive \(\tau_i\) or \(\tau_d\) are disabled in code. The derivative branch is filtered by \(N_d\) to avoid ideal high-frequency differentiator gain.

the reduced closed-loop matrix is:

\[ M_{\mathrm{closed}}(\omega)=M_{\mathrm{open}}(\omega)-B_xH_x(\omega)C_\phi. \]

This keeps only first-order terms: \(dx\), \(\phi\), and \(H_x\phi\) are retained, while products such as \(dx\,\phi\), \(dx\,r\), and \(dx\,T_1\) are outside the linear model.

This is the exact matrix structure currently implemented in Sensor.m_matrix().

Augmented \(dx\) System

For the controller derivation, \(dx\) is kept as an independent variable until the feedback equation is added. The first-order expansion around the operating detuning \(x_0\) is:

\[ M_{\mathrm{open}}(\omega,x_0)Y-B_x(x_0)dx=N,\qquad Y=\begin{bmatrix}r&\phi&T_1\end{bmatrix}^{T}. \]

The fourth equation introduces the controller after the sideband variables have been transformed into demodulated normalized coordinates:

\[ dx-H_x(\omega)C_\phi Y=0. \]

The frequency-domain system inverted by the code is therefore:

\[ \begin{bmatrix} M_{\mathrm{open}}(\omega,x_0) & -B_x(x_0)\\ -H_x(\omega)C_\phi & 1 \end{bmatrix} \begin{bmatrix} Y\\ dx \end{bmatrix} = \begin{bmatrix} N\\ 0 \end{bmatrix}. \]

When PID is off, \(H_x=0\). The fourth equation gives \(dx=0\), and the first three equations reduce exactly to the paper matrix \(M_{\mathrm{open}}(\omega,x_0)Y=N\).

Eliminating \(dx\) with a Schur complement gives the reduced matrix above, so the \(Y\) response from the augmented inverse is identical to \(Y=M_{\mathrm{closed}}^{-1}N\). With \(H_x=0\), this inverse reduces to the paper form \(Y=M_{\mathrm{open}}^{-1}N\) and a solved \(dx=0\).

Python: Sensor.m_matrix_augmented_array() builds this matrix, and frequency-domain noise/source solves use the augmented inverse.

Connection to \(M_t\) and Stability

Time-domain linear dynamics are obtained from \(M(\omega)\) using:

\[ D_0=M(0),\qquad D_1=-i\,\frac{\partial M}{\partial\omega},\qquad M_t=-D_1^{-1}D_0. \]

See Mt Stability for eigenvalue checks and stability criteria.

Scope

This matrix model is a small-signal linearization about an operating point. Large-signal nonlinear behavior requires a nonlinear time-domain model.
